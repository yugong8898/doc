## 一、定位

技术雷达用于沉淀前端、AI Coding、开发工具链、工程化、监控、埋点等外部信息，帮助 WLYD 项目持续关注可落地的新工具、新方法和新风险。

本目录与 `.repoWiki/` 分工如下：

| 目录 | 定位 | 内容边界 |
|------|------|----------|
| `.repoWiki/` | 项目内部知识库 | WLYD 现有源码语义索引、业务规则、坑点、交接信息 |
| `docs/tech-radar/` | 外部技术雷达 | 外部工具、行业动态、AI/前端趋势、可引入方案评估 |

角色和 Prompt 仍以 `.md/prompts/` 目录维护内容为准；本目录只沉淀资讯、工具评估和筛选规则，不替代角色体系。

## 四、更新节奏

试运行规则：

- 执行时间：工作日上午 10:00
- 试运行周期：先执行 3 个工作日
- 输出方式：只生成 Markdown，不自动提交 Git
- 复盘方式：三次结果出来后，按内容质量调整来源、关键词、排除规则，再决定是否长期保留自动任务

## 五、结果筛选原则

优先保留：

1. 能直接改善 WLYD 开发流程的工具或方法。
2. 对 Vue2 存量项目迁移、兼容、质量提升有帮助的内容。
3. 对 Cursor / Qoder / CodeBuddy / Agent 工作流有启发的内容。
4. 能补充 `.repoWiki/`、Skills、Prompts 的知识管理方法。
5. 与前端监控、埋点、质量、安全相关的可落地实践。

优先排除：

1. 纯营销软文，无可验证实践。
2. 只适用于后端或云原生，与前端关系弱的内容。
3. 与当前技术栈冲突且短期无法试用的重型方案。
4. 重复信息、标题党、低质量转载。

详细规则见 [排除规则](./exclude-rules.md)。

## 六、使用方式

### 1. 查看每日结果

打开 [每日技术雷达索引](./daily/README.md)，再进入对应月份文档。

### 2. 调整关注范围

编辑 [关注主题与关键词](./topics.md)，新增或降低关键词优先级。

### 3. 调整来源质量

编辑 [来源清单](./sources.md)，将低质量来源移入降权或排除列表。

### 4. 调整排除规则

编辑 [排除规则](./exclude-rules.md)，补充标题关键词、内容关键词或来源规则。

## 七、后续优化方向

- 根据三天试运行结果建立精选规则。
- 将高价值内容转入 `.repoWiki/` 或 `.cursor/context/known-pitfalls.md`，但必须先确认内容确实适用于 WLYD。
- 若稳定可用，再考虑自动提交 Git 或自动生成周报摘要。
