# wlyd-app-consigner 项目总览

## 项目概述

**项目名称**: ==万连通货主端 (wlyd-app-consigner)==
**项目类型**: 基于 uni-app 的跨端移动应用  
**支持平台**: 微信小程序 / Android APP / ~~iOS APP~~ / H5

货主端是万连通物流平台的货主侧移动应用，提供发货下单、订单管理、运单跟踪、结算处理等核心功能。

## 路由机制

项目不依赖第三方路由库，使用 `uni.addInterceptor` 拦截原生跳转方法实现路由守卫。

**`hooks/useRouter.js`** — 核心 Router 类，通过拦截 uni 的以下 5 个跳转 API 实现全局守卫：
- `navigateTo` / `redirectTo` / `reLaunch` / `switchTab` / `navigateBack`

**`router/index.js`** — 注册守卫回调：
- `beforeEach`：当前为空，可用于鉴权拦截
- `afterEach`：页面跳转完成后触发海纳嗨数埋点 `H_AppViewScreen`（仅 APP 端）

**`util/navigation.js`** — 统一导航工具，封装常用跳转场景：

| 方法 | 说明 |
|------|------|
| `navigateToWebView(url, params)` | 跳转到内嵌 WebView 页，自动拼接 token 及系统参数 |
| `navigateToMiddle(token, path)` | 跳转中间页（用于 SSO 跳转） |
| `goToSendGoods()` | 跳转发货页（H5 环境下通过 postMessage 通知宿主） |
| `goToProtocolRule()` | 跳转协议规则页（带埋点） |
| `switchPlatform(url)` | 切换平台 |
| `navigate(path, options)` | 通用跳转，支持 navigateTo / switchTab / reLaunch 等 |

## HTTP 请求

基于 `@wlydfe/uni_http`（uView 的 http 模块）封装，挂载在 `uni.$u.http`。

### 初始化流程

```
config/request.js
  ├── setConfig: baseURL = config.baseUrl
  ├── require('./requestInterceptors')   # 请求拦截器
  └── require('./responseInterceptors')  # 响应拦截器
```

### 请求拦截器 (`requestInterceptors.js`)

- 确保 `config.data` 不为 undefined（赋默认 `{}`）

### 响应拦截器 (`responseInterceptors.js`)

- `code !== 200` 时默认弹出 `uni.$u.toast(data.message)`
- 支持 `custom.toast = false` 关闭默认提示
- 支持 `custom.catch = true` 将错误 reject 出去（否则返回 pending Promise，静默失败）
- 成功时直接返回 `data.data`

### 使用示例

```js
// GET（错误已被拦截器统一处理，无需 catch）
uni.$u.http.get(url, { params })

// POST，需要自行处理错误
uni.$u.http.post(url, body, { custom: { catch: true } }).catch(err => {})
```

## 权限系统

**`mixins/auth.js`** 以全局 mixin 方式注入，提供两种权限验证方式：

**1. 全局函数 `uni.$authCheck(authItem)`**  
用于命令式权限校验（如按钮点击前），从 `app.globalData.authList` 读取权限列表。

```js
if (!uni.$authCheck({ auth: '10001', tip: '暂无操作权限' })) return
```

**2. Mixin 方法**

| 方法 | 说明 |
|------|------|
| `exLoadMenu()` | 加载并缓存菜单权限 |
| `exIsHasPageAuth(authEnum)` | 校验页面是否有任一权限，无权限时 toast 提示并延迟 resolve(false) |
| `computed.getAuthList` | 从 store 获取权限列表 |

**海纳嗨数 (HinaData)**

- `util/umtrack.js` 封装，用于 APP 渠道统计

## 全局组件注入

通过 `custom-vue-inset-loader` 在 `pages.json` 的 `insetLoader` 配置中自动向所有页面注入：

```json
{
  "insetLoader": {
    "config": {
      "wToast": "<wToast ref='wToast'></wToast>"
    },
    "label": ["wToast"],
    "rootEle": "view"
  }
}
```

注入后任意页面均可通过 `this.$refs.wToast` 调用 Toast。

## 多平台差异处理

项目大量使用 uni-app 条件编译处理平台差异：

| 场景 | APP 端 | 小程序端 | H5 端 |
|------|--------|----------|-------|
| 路由埋点 | 触发（取 webview title） | 不触发 | 不触发 |
| 菜单权限字段 | `funcId` | `funcId` | `isOwnerH5=true` 时取 `item2` |
| 客户端类型值 | `10` | `20` | `40` |
| WebView 跳转 | 直接跳转 | 直接跳转 | postMessage 通知宿主 |
| 结算 Tab | 显示 | 显示 | 隐藏 |

## 打包发布

### H5 打包
1. 修改 `config/env.js` 中 `preFix = location.origin`
2. 确认 `manifest.json` 中 `h5.router.mode = hash`，`h5.router.base = /yw/m/uniapp-oner/`
3. HBuilderX → 发行 → 网站-H5手机版
4. 将产物替换到 `logisticsweb/public/m/uniapp-oner/` 目录

### Android APP 打包
1. 恢复 `config/env.js` 为 APP 配置（`preFix = uni.getStorageSync(...)`）
2. HBuilderX → 发行 → 原生App-云打包
3. 包名：`com.wanlianyida.consignor.app`，证书别名：`owneralias`

### 微信小程序发布
1. 合并开发分支到 `develop`
2. HBuilderX → 发行 → 小程序-微信
3. 在微信公众平台设置体验版

### 分支规范
- `master`：生产
- `release`：预发布
- `develop`：开发/小程序
- `F-xxx`：功能分支
