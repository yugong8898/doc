# logisticsweb 项目总览

## 概述

`logisticsweb` 是**万联易达无车承运人平台（PC 端 3PL）**的前端项目，基于 Vue 2 + Element UI 构建。

项目同时支持两种运行模式：
- **独立运行**：作为独立的 Vue SPA 项目运行
- **qiankun 子应用**：接入 `match-owner-pc` 主应用（app-main）的微前端体系，挂载在 `/app-3pl/` 路径下

## 核心逻辑

### 1. 应用启动流程

```
main.js
  ├── 独立运行: render() → handleUrlParams() → import('./router/index') → initVueApp()
  └── qiankun 子应用:
        bootstrap() → mount(props) → 同步 token + 用户信息 → render(props)
                                                                     ↓
                                                          unmount() → 销毁实例 + 取消状态订阅
```

关键实现位置：`src/main.js`

- `render()` 负责初始化 Vue 实例，注入路由和 Vuex store
- `mount(props)` 在 qiankun 模式下从主应用 `props.userStore` 同步用户信息和 token，再调用 `render(props)`
- `unmount()` 销毁 Vue 实例并恢复 `appendChild` 方法（`microAppDialog` 挂载处理）

### 2. 路由系统

**静态路由**（`src/router/routers.js`）：

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | redirect | 重定向到 `/systemlogin` |
| `/systemlogin` | `views/login` | 登录页 |
| `/login` | `views/unifiedLoginPage` | 统一门户登录页 |
| `/findpwd` | `views/findpwd` | 找回密码 |
| `/register` | `views/register` | 注册 |
| `/authorization` | `views/register/authorization` | OAuth 授权页（无需登录） |
| `/accountBinding` | `views/userBase/modifypwd/AccountBinding` | 账号绑定 |
| `/404` | `views/404` | 404 页面 |

**动态路由**（`src/router/index.js`）通过路由守卫 `beforeEach` 实现：

```
用户访问 userBase/** 路径
    ↓
getToken() 有效？
    ├── 是 → bool === false? → loadMenus() → 拉取菜单/权限 → filterAsyncRouter() → router.addRoutes()
    └── 否 → notifyNeedRelogin() → 跳转登录页
```

`filterAsyncRouter()` 根据后端返回的菜单数据动态生成路由，所有业务页面统一挂载在 `/userBase` 下，组件路径映射为 `@/views/userBase/{路径}`。

**微前端路由**（`src/router/micro-router.js`）：当 qiankun 主应用下发路由配置时，通过 `createRouter()` 动态创建路由实例。

### 3. 权限控制

- **页面权限**：通过动态路由实现，未分配菜单的路由无法访问
- **按钮权限**：`filterAsyncRouter()` 将按钮权限（`funcType === 2`）注入路由 `meta`，页面通过 `$store.state.app.controlBtn` 读取
- **组织机构权限**：`organizationControl` 单独存储 workspace 页面的权限 meta

读取示例：
```js
computed: {
  controlBtn() {
    return this.$store.state.app.controlBtn
  }
}
```

### 4. 请求拦截（`src/utils/request.js`）

基于 axios 封装，核心处理逻辑：

**请求拦截器**：
- 检测 token 变更（多设备登录场景），触发强制登出
- 微前端模式下，通过 `__tabTokenCache__` 检测跨 Tab 账号切换
- 自动添加 `Authorization` 和 `Systype` 请求头
- 通过 `addHttpCommonData()` 注入公共参数（`passport`、`client_biz_type`）

**响应拦截器**：
- `succeed/succ === true` → 正常返回数据
- 错误码命中 `WLYD_ERR_CODE` → 触发登出并跳转登录页
- `config.responseType === 'blob'` → 直接返回（用于文件下载）
- `config.method === 'get'` → 不做错误拦截（导出接口均为 GET）

**token 失效处理**：
- 独立模式：`store.dispatch('app/doLogOut')` + 跳转登录
- qiankun 子应用：调用 `getMainUserStore().doLogout()` 委托主应用处理

### 5. Token 管理（`src/utils/auth.js`）

| 函数 | 说明 |
|------|------|
| `getToken()` | 优先读 `Vue.prototype.$getToken()`（qiankun 注入），降级读 Cookie |
| `setToken(token, rememberMe)` | 写入 `SSO-TOKEN` Cookie，设置跨子域 domain |
| `removeToken()` | 删除 Cookie |
| `setSwitchCompanyId()` | 切换业务公司时存储 companyId |
| `syncTokenCache()` | 主动同步 tokenCache，避免切换企业误判为多设备登录 |

### 6. 布局系统（`src/layout/`）

```
Layout.vue（主布局容器）
  ├── Header.vue          # 顶部导航栏（isEmbedMode 时隐藏）
  ├── leftFirstLevelMenu  # 左侧一级菜单（isEmbedMode 时隐藏）
  ├── Sidebar.vue         # 左侧二级菜单（menuCollapse 时折叠）
  ├── AppMain.vue         # 右侧主内容区（router-view）
  └── Bottom.vue          # 底部栏（isEmbedMode 时隐藏）
```

`isEmbedMode`：通过 URL 参数 `?embed=true` 触发，隐藏所有导航 UI，仅显示内容区，用于嵌入其他系统。

## API 接口文件规范

`src/api/` 目录命名规则：

- `userBase.{模块名}.js` — 用户中心各业务模块接口
- `login.js` — 登录相关
- `public.js` — 公共接口（城市数据、友情链接等）
- `home.js` — 首页/工作台接口
- `common.js` — 通用接口

## 状态管理数据流

```
路由守卫 beforeEach
    ↓
app/loadMenus (获取用户信息 + 菜单 + 权限)
    ↓
filterAsyncRouter (生成路由 + 注入按钮权限到 meta)
    ↓
router.addRoutes (注册动态路由)
    ↓
afterEach: app/SET_CONTROL_BTN (同步当前路由 meta 到 store)
    ↓
页面组件通过 $store.state.app.controlBtn 读取权限
```

## 微前端集成要点

独立运行 vs qiankun 子应用的差异：

| 特性 | 独立运行 | qiankun 子应用 |
|------|---------|--------------|
| publicPath | `/yw/` | 动态（`__webpack_public_path__`） |
| 路由 base | `/yw/` | `/app-3pl/` |
| Token 来源 | Cookie `SSO-TOKEN` | 主应用 `props.getToken()` |
| 用户信息 | 自行请求接口 | 从 `props.userStore` 同步 |
| 登出处理 | 自行调用登出接口 | 委托主应用 `doLogout()` |
| 弹窗挂载 | 默认 body | `initMicroAppDialog()` 处理挂载容器 |

## 注意事项

1. **路由跳转前缀**：独立运行用 `/yw/{路由}`，qiankun 子应用用 `/app-3pl/{路由}`
2. **按钮权限读取**：必须在 `activated` 或 `mounted` 中遍历 `$route.meta` 或读取 `$store.state.app.controlBtn`
3. **多设备登录检测**：request.js 会检测 tokenCache 与当前 token 是否一致，切换企业时需调用 `syncTokenCache()` 避免误触发
4. **嵌入模式**：URL 带 `?embed=true` 时隐藏全部导航，适用于被其他系统 iframe 嵌入的场景
5. **ESLint**：项目使用 `airbnb` + `vue/recommended` 规则，无分号，单引号，2 空格缩进
