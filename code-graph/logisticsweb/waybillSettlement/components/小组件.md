## 一、WeighBillCarousel.vue — 磅单图片轮播

### 概述

根据 `waybillId + transportType + type`（装货/卸货）自动拉取磅单图片，展示轮播缩略图，点击可全屏预览并支持旋转保存。

仅 `transportType ∈ {1, 2, 3, 6}` 的运输类型有磅单数据，其他类型展示空占位图。

### Props

| prop | 类型 | 说明 |
|------|------|------|
| `type` | String | `'load'` 装货 / `'unload'` 卸货 |
| `waybillId` | String | 运单 ID |
| `transportType` | String/Number | 运输类型，决定查询用的 `wabillStatus` |
| `showDownBtn` | Boolean | 全屏预览时是否显示下载按钮 |

### 内部常量

```js
// 各运输类型对应的 wabillStatus
WEIGH_BILL_STATUS_MAP = {
  '1': { load: '410', unload: '430' },
  '2': { load: '410', unload: '430' },
  '3': { load: '410', unload: '420' },
  '6': { load: '410', unload: '420' }
}
// 附件类型：1=装货磅单，2=卸货磅单
WEIGH_BILL_ATTACHMENT_TYPE = { load: ['1'], unload: ['2'] }
```

### 核心逻辑

```
getBillImages(waybillId)
  → 查 WEIGH_BILL_STATUS_MAP，无对应 transportType 则清空并返回
  → fetchWeighBillImages({ waybillId, status, attachmentTypes })
  → resolveAttachmentItems(rawItems, ossKey => getReceiptUrl({ path: ossKey }))
    （blob URL，旋转保存后重新拉图以同步更新缩略图）
  → syncImgList() 同步 imgList / imgListUrl / saveImgListUrl
```

旋转保存后触发 `handleImageSaved()` 重新拉图，缩略图同步更新。`saveImgListUrl` 保存 OSS 原始路径供后端"记住旋转方向"接口使用。

### 调用接口

| 接口 | 说明 |
|------|------|
| `fetchWeighBillImages` | `userBase.transport` - 查询磅单附件列表 |
| `getReceiptUrl` | `userBase.paymentTob` - 获取 OSS 访问 URL（blob） |

## 二、AnchorNav.vue — 锚点导航

### 概述

结算信息页顶部固定的分区快速定位导航，点击导航项平滑滚动到对应区块。

### Props

| prop | 类型 | 说明 |
|------|------|------|
| `list` | Array | 导航项列表，每项 `{ id: string, label: string }`；`id` 对应页面元素的 DOM id |

### 定位逻辑

```
scrollToAnchor(id)
  → activeAnchor = id
  → getScrollParent(el.parentElement)  向上递归找最近的 overflow:auto/scroll 容器
  → getOffsetToContainer(el, container) 累加 offsetTop 得到容器内绝对偏移
  → container.scrollTop = offset - 40 - 60 - 20
    （40: 导航栏高度；60: 页面顶部头高度；20: 视觉留白）
  → 无可滚动容器时退回 el.scrollIntoView({ block: 'start' })
```

### 吸顶样式

由 `anchorNavStyle` computed 根据 `isMicroApp()` 自动计算：

| 环境 | top | left |
|------|-----|------|
| 主应用 | 60px | 246px（sidebar 108+123+15） |
| 子应用 | 50px | 210px |

### 使用示例（edit.vue / read.vue）

```html
<AnchorNav :list="anchorNavList" />
```

```js
// 锚点定义（anchorNavList 由 computed 动态插入风控锚点）
[
  { id: 'anchorWaybillInfo',     label: '运单信息' },
  { id: 'anchorSettlementInfo',  label: '结算信息' },
  { id: 'anchorCostDetail',      label: '费用明细' },
  { id: 'anchorRiskWarning',     label: '风控预警' },  // showRiskWarning=true 时插入
  { id: 'anchorExamResult',      label: '预审结果' },
  { id: 'anchorOperationRecord', label: '操作记录' }
]
```

## 三、rushIcon.vue — 催付图标

### 概述

列表页运单号列旁的催付次数角标，`value > 0` 时显示。

### Props

| prop | 类型 | 说明 |
|------|------|------|
| `value` | Number（必填） | 催付次数；`= 0` 时不显示 |

### 展示规则

- `value >= 100`：角标显示 `99+`，否则显示实际次数
- tooltip：`司机已催运费 N 次`
- 图标样式：蓝底白字"催"字方块，右侧红色角标

## 四、loseDialog.vue — 亏吨货损确认弹窗

### 概述

在调整结算信息页（edit.vue）中，当 `cargoCompensationConfirmFlag === '1'`（待处理）时自动弹出，用户填写货损单价和重量后提交确认。

### Props

| prop | 类型 | 说明 |
|------|------|------|
| `showDialog` | Boolean | 弹窗显示状态，`.sync` 绑定 |
| `row` | Object | 运单行数据 |
| `bankOptions` | Object | 开户行字典 |
| `loseDetails` | Object | 亏涨吨展示数据（传给内部 `LoseAndRiseViewText`） |
| `lsdsGoodsDeductible` | Object | 亏吨货损免赔配置，`watch` 后回填 form |
| `initTransportCost` | String | 运费金额（货损金额不得超过运费） |
| `settlementWeight` | String/Number | 结算重量（货损重量不得超过结算重量） |

### 表单字段

| 字段 | 校验规则 |
|------|---------|
| `goodsLostValue` 货损单价（元/吨） | 必填；`numSevenCodeTwo`（整数7位，小数2位） |
| `lossWeight` 亏吨货损重量（千克） | 必填；`numFiveCodeThree`；不得 > 结算重量 |
| `lossCast` 亏吨货损金额（元） | 必填；不得 > 运费 |

### 联动计算

```
watch goodsLostValue / lossWeight
  → lossCast = goodsLostValue × (lossWeight / 1000)
  → clearValidate(['lossCast'])
```

### Events

| 事件 | 触发时机 | 参数 |
|------|---------|------|
| `closeDialog` | 点击取消 | 无 |
| `closeDialog(params)` | 点击确定且校验通过 | `{ goodsLostValue, lossWeight, lossCast }` |

## 五、examResult.vue — 预审结果展示

### 概述

纯展示组件，展示运单预审结果和不通过原因。

### Props

| prop | 类型 | 说明 |
|------|------|------|
| `preExaminationReason` | String | 预审不通过原因；空字符串时显示"通过"；有值时显示"不通过"及原因文本 |

### 展示逻辑

```
preExaminationReason 有值（非空非null）  → 预审结果：不通过；显示不通过原因文本框
preExaminationReason === ''             → 预审结果：通过；不显示原因
```

## 六、warnTable.vue — 运单状态异常警告弹窗

### 概述

在批量提交运单时，`checkOrderStatus` 接口返回有问题的运单列表时弹出，展示德邦平台状态异常的运单，支持手动刷新。

### 内部数据

| 字段 | 说明 |
|------|------|
| `tableData` | 展示的运单列表 |
| `loading` | 刷新时的 loading 状态 |

### 表格列

| 列 | prop | 说明 |
|----|------|------|
| 平台运单号 | `waybillId` | show-overflow-tooltip |
| 德邦平台订单号 | `orderCode` | show-overflow-tooltip |
| 德邦平台运单状态 | `orderStatusName` | show-overflow-tooltip |

### Events

| 事件 | 触发时机 |
|------|---------|
| `refresh` | 点击刷新按钮，父组件重新调 `checkOrderStatus` |

### 对外方法（通过 ref 调用）

```js
this.$refs.warnTable.handleOpenDailog(data)  // 打开弹窗并传入数据
this.$refs.warnTable.handleClose()           // 关闭弹窗
```

## 七、editLoadWeight.vue — 修改装卸重量弹窗

### 概述

在调整结算信息页（edit.vue）中，已上报且未结算通过的运单可通过此弹窗修改装卸净重和毛重。

### Props

| prop | 类型 | 说明 |
|------|------|------|
| `visible` | Boolean | 弹窗显示状态；watch immediate 同步 `dialogVisible` 并回填表单 |
| `loading` | Boolean | 提交按钮 loading 状态 |
| `formData` | Object | 当前重量数据，弹窗打开时回填 |

### 表单字段

| 字段 | 是否必填 | 校验规则 |
|------|---------|---------|
| `loadWeight` 装货净重（吨） | 必填 | `numSevenCodeFour`（整数7位，小数4位） |
| `loadRoughGoodsWeight` 装货毛重（吨） | 非必填 | 同上（有值时才校验格式） |
| `unloadWeight` 卸货净重（吨） | 必填 | 同上 |
| `unloadRoughGoodsWeight` 卸货毛重（吨） | 非必填 | 同上 |

### Events

| 事件 | 触发时机 | 参数 |
|------|---------|------|
| `close` | 点击取消 | 无 |
| `confirm` | 点击提交且校验通过 | `{ loadWeight, loadRoughGoodsWeight, unloadWeight, unloadRoughGoodsWeight }` |

### 父组件处理（edit.vue）

```js
handleConfirmEditLoadWeight(formData)
  → updateWeight({ waybillId, versionCode, ...formData })  （versionCode 乐观锁）
  → 若已确认货损（cargoCompensationConfirmFlag='2'）且重量有变化 → 弹提示
  → 成功后刷新页面数据
```
