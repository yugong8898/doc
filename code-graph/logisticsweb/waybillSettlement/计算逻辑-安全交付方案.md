# 运单结算计算逻辑 — 安全交付方案（原子级拆分）

> 核心原则：每个 PR 只做一件事，改完当天能测完，出问题可秒回滚。
> 所有改动不改变外部行为（接口调用参数不变、用户操作结果不变），只改内部实现。

---

## 风险控制策略

| 策略 | 说明 |
|------|------|
| 单文件单 PR | 每个 PR 只改 1 个文件，revert 成本为 1 个 commit |
| 行为不变优先 | 先做"不改变任何计算结果"的重构，再做 bug 修复 |
| 新旧并行 | 关键改动保留旧代码（注释），灰度验证后再删 |
| 测试前置 | 每个 PR 附带手动验证 checklist，测试通过才合 |
| 渐进替换 | 不一次性替换所有调用点，一个个替换并验证 |

## PR-2：油气优惠接口加 debounce + 请求竞态保护

**改 1 个文件**：`CostMessageDetails.vue`
**改动量**：约 15 行

**改什么**：
```js
// 改前：updBusCostBlur 直接调用
await this.updateOilGasDiscountIfNeeded()

// 改后：
// 1. data 里加 _oilGasRequestId: 0
// 2. updateOilGasDiscountIfNeeded 改为 debounce 包裹
updateOilGasDiscountIfNeeded: debounce(async function() {
  if (this.detailObj.fullProcessFlag === 10) return
  const oilAmount = this.detailObj?.oilAmount || this.detailObj?.bmsWaybillPoolExt?.oilAmount
  if (!oilAmount || Number(oilAmount) <= 0) return

  const requestId = ++this._oilGasRequestId
  // ... 原有接口调用逻辑不变 ...
  
  // 接口返回后检查是否过期
  if (requestId !== this._oilGasRequestId) return
  // ... 原有更新逻辑不变 ...
}, 500)
```

**为什么安全**：
- 计算结果不变，只是减少了调用频率
- 最终一定会调一次（debounce 保证最后一次执行）
- 全流程跳过调用只是避免无效请求（原来调了接口结果也展示 0）

**验证 checklist**：
```
□ 修改 1 行确认金额后 blur：油气优惠行正常更新
□ 连续修改 3 行后 blur：只有最后一次触发接口调用（Network tab 验证）
□ 全流程运单：油气优惠始终为 0，无接口调用
□ 无油气费运单：不触发接口调用
```

**回滚方式**：revert 该 commit

## PR-3：小计计算与 handleGetSummaries 分离

**改 1 个文件**：`CostMessageDetails.vue`

**改什么**：

新增方法：

```js
calcSubtotal() {
  const sum = this.params.model.reduce((total, item) => {
    if ([101, 102].includes(item.orderNumber)) return total
    if (['200', '600'].includes(`${item.costType}`)) {
      return total.plus(+(item.updBusCost || 0))
    } else {
      return total.minus(+(item.updBusCost || 0))
    }
  }, new BigNumber(0))
  this.subtotal = Number(sum)
}
```

```js
} else if (column.property === 'updBusCost') {

  const sum = new BigNumber(this.subtotal)
  // ... 展示逻辑不变 ...
}
```

在触发点显式调用：
```js

dialogConfirmZeroInfo(diffAmount, processedTotalAmount) {
```

## PR-5：getDetails loading 在 finally 重置

```js
if (datas) {
  this.loading = false
  // ...
} else {
  this.loading = false
}
// catch 分支没有重置！

// 改后：用 try/finally 包裹
async getDetails() {
  if (!this.waybillPoolId) { ... }
  if (this.loading) return
  this.loading = true
  try {
    // ... 原有全部逻辑（if (datas) ... else ...）...
  } finally {
```

- finally 保证无论如何都重置

## PR-8：提取纯计算函数到 utils/calculation.js

**改动文件**：新建 `utils/calculation.js` + 修改 `CostMessageDetails.vue`
**改动量**：新建约 30 行 + 修改约 10 行

**改什么**：

新建文件（纯函数，无副作用）：
```js
// src/utils/calculation.js
import BigNumber from 'bignumber.js'

export function getBit(value, n) {
  let str = value.toFixed(n + 1).toString()
  str = str.split('.')[0] + '.' + str.split('.')[1].substring(0, n)
  return parseFloat(str)
}

export function formatService(num, n, type) {
  const typeObj = { '0': Math.ceil, '1': Math.floor, '4': Math.round }
  const fn = typeObj[type] || Math.ceil
  const newNum = fn(new BigNumber(num).multipliedBy(Math.pow(10, n)).toNumber())
  return new BigNumber(newNum).div(Math.pow(10, n)).toNumber()
}

export function calcTotal(amount, rate, roundingMode) {
  const divisor = getBit(1 - (parseFloat(rate) / 100), 7)
  let num = new BigNumber(amount).dividedBy(divisor).toFixed(7)
  return +formatService(+num, 2, roundingMode || 0)
}
```

CostMessageDetails 改为 import：
```js
import { calcTotal, formatService, getBit } from '@/utils/calculation'

// count 方法改为：
count(Amount, Rate) {
  return calcTotal(Amount, Rate, this.roundingMode)
}
```

**为什么安全**：
- 纯函数提取，计算逻辑 100% 不变
- CostMessageDetails 的 `count` 方法保留（作为薄代理），对外接口不变
- `endOfCalculation.js` 暂时不改（后续单独 PR 替换）

**验证 checklist**：
```
□ 运费差价计算结果与改动前一致（对比 3 张不同运单的数值）
□ 不同 roundingMode（0/1/4）下结果一致
□ 差价率为 0 时不报错
□ 金额为 0 时不报错
```

**回滚方式**：revert 该 commit

## 灰度验证策略

每个 PR 合入后观察周期：

| 批次 | 观察期 | 观察内容 |
|------|--------|---------|
| 第 1 批 | 1~2 天 | 线上无新增报错、运单保存/提交正常 |
| 第 2 批 | 2~3 天 | 费用明细计算正确、抹零金额正确 |
| 第 3 批 | 3~5 天 | 各类运单（普通/B1/全流程/POW）均正常 |

**出问题的应急方案**：单个 PR revert，不影响其他已合入的改动（因为互相独立）。
