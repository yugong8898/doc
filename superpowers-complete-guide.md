## ⚡️ 核心功能详解

### 1. Git Blame (责任归属)

**功能**: 每行代码旁显示最后修改者和时间

```vue
303 │ <div v-if="!isInMicroApp">  👤 王新骏 4小时前
304 │   <div class="credit-account"> 👤 张三 2天前
305 │     <span>授信总额</span>      👤 李四 1周前
```

**用途**:
- ✅ 快速找到代码责任人
- ✅ 理解代码编写时间和背景
- ✅ 追踪 Bug 引入时间

---

### 2. File History (文件历史)

**功能**: 时间线显示文件所有修改

```
wallet/mine.vue 修改历史:
├─ 4h ago  环境判断 (王新骏) +2 -2
├─ 5h ago  undefined修复 (王新骏) +5 -3
├─ 8h ago  样式问题 (王新骏) +10 -8
├─ 1d ago  UI调整 (王新骏) +25 -20
├─ 2d ago  更换接口 (王新骏) +45 -30
└─ ...查看更多
```

**用途**:
- ✅ 理解代码演进过程
- ✅ 回溯历史版本
- ✅ 分析修改频率和风险

---

### 3. Commit Graph (提交图表)

**功能**: 可视化分支结构和合并历史

```
      develop-delta
     ╱│╲
    ╱ │ ╲
   🟢 │  🟡 C103.3 钱包优化 (你的分支)
   │  │   ├─ 12 commits
   │  │   └─ ✅ 已合并
   │  │
   │  🟡 C104.1 货物表单重构
   │     └─ 无冲突
   │
   🟡 结算发票迁移
      └─ ⚠️ 注意影响范围
```

**用途**:
- ✅ 理解复杂的分支关系
- ✅ 检测潜在冲突
- ✅ 追踪功能合并状态

---

### 4. Compare Branches (分支对比)

**功能**: 对比两个分支的差异

```
develop-delta ↔️ F-C.103.3-tem

📊 差异统计:
├─ 新增文件: 0
├─ 修改文件: 1 (mine.vue)
├─ 删除文件: 0
└─ 总改动: +79 -81 行

🔍 详细对比:
mine.vue:
  - 环境判断结构调整
  - UI 样式优化
  - undefined 处理
```

**用途**:
- ✅ 准备合并前检查
- ✅ 理解分支差异
- ✅ 评估合并风险

---

### 5. Pull Request 管理

**功能**: 在 IDE 内完成 PR 全流程

```
创建 PR:
1. 右键分支 → Create Pull Request
2. 自动填充标题、描述、标签
3. 选择审查者
4. 一键创建 ✅

审查 PR:
1. 查看 PR 列表
2. 点击打开详情
3. 逐文件审查改动
4. 添加行内评论
5. 批准或请求修改 ✅
```

**用途**:
- ✅ 无需切换浏览器
- ✅ 更快的审查流程
- ✅ 上下文不中断

### Code Review 检查清单

使用 GitLens 审查 PR 时：

#### ✅ 提交质量
- [ ] 提交信息清晰描述改动
- [ ] 提交历史合理，无调试信息
- [ ] 相关改动合并在一个提交

#### ✅ 代码改动
- [ ] 改动符合需求描述
- [ ] 没有无关文件的修改
- [ ] 代码风格保持一致

#### ✅ 影响范围
- [ ] 使用 Compare 查看完整差异
- [ ] 检查是否影响其他模块
- [ ] 评估潜在的回归风险

#### ✅ 协作情况
- [ ] 查看 File History 了解背景
- [ ] 检查是否与他人改动冲突
- [ ] 确认改动不覆盖他人工作

### 团队协作建议

#### 1. 建立分支命名规范
```
feature/C{需求号}-{功能简称}-{姓名拼音}
例如: F-C.103.3-wallet-wxj
```

#### 2. 使用一致的提交信息格式
```
feat: 【C103.3】功能描述 #CHJY-20364
fix: 修复xxx问题 #CHJY-20365
refactor: 重构xxx模块 #CHJY-20366
```

#### 3. 定期同步主分支
```
使用 GitLens:
1. 右键 develop-delta
2. 选择 "Pull"
3. 自动合并到当前分支
```

#### 4. 及时解决冲突
```
GitLens 冲突可视化:
├─ 左侧: 你的改动
├─ 右侧: 他们的改动
└─ 点击选择保留版本 ✅
```
