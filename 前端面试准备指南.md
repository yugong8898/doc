# 前端面试准备指南

## 一、技术提升路线图

### 1.1 深度：框架底层原理

#### React 方向
- **React 18+ 新特性**：Concurrent Mode、Suspense、Server Components
- **源码级理解**：Fiber 架构、调和算法、Hooks 实现原理
- **性能优化**：虚拟 DOM diff、批量更新、时间切片
- **状态管理演进**：Redux → Zustand/Jotai/Recoil 原理对比

#### Vue 方向
- **Vue 3 Composition API** 深度应用
- **响应式原理**：Proxy vs defineProperty
- **编译优化**：静态提升、Patch Flag

#### 实践建议
- 手写简易版 React/Vue（理解核心机制）
- 分析框架源码（至少一个核心模块）
- 写技术博客输出理解

### 1.2 广度：技术栈扩展

#### 工程化能力
- **构建工具**：深入 Vite/Webpack/Rspack 配置与优化
- **微前端**：qiankun/Micro-App 实践
- **Monorepo**：pnpm workspace + Turborepo/Nx
- **CLI 工具开发**：脚手架、代码生成器

#### 全栈延伸
- **Node.js**：Express/Koa/Nest.js，理解服务端思维
- **TypeScript**：高级类型、泛型、装饰器深度应用
- **GraphQL**：替代 REST API 的方案
- **SSR/SSG**：Next.js/Nuxt.js 实战

#### 新技术探索
- **Rust 系工具链**：SWC、Turbopack
- **WebAssembly**：性能密集型场景
- **Web Components**：跨框架组件方案

### 1.3 专业领域深耕

选择 1-2 个方向成为专家：

#### 性能优化
- 首屏加载优化（资源分割、预加载、骨架屏）
- 运行时性能（虚拟列表、Web Worker、requestIdleCallback）
- 监控体系：Performance API、Sentry、自建监控
- 性能指标：LCP、FID、CLS 优化实践

#### 可视化/交互
- Canvas/SVG/WebGL 深度应用
- Three.js/D3.js/Echarts 高级定制
- 动画库：Framer Motion/GSAP
- 低代码平台开发

#### 工程架构
- 组件库设计与开发（从 0 到 1）
- 设计系统（Design System）
- 前端监控/埋点体系
- CI/CD 流程设计

### 1.4 软实力提升

#### 代码质量
- 设计模式在前端的应用
- 重构技巧
- 单元测试：Jest/Vitest + Testing Library
- Code Review 能力

#### 项目管理
- 需求分析与技术方案设计
- 风险评估与影响范围分析
- 技术债务管理
- 团队协作与沟通

#### 学习方法
- 输出驱动：技术博客、开源项目、技术分享
- 源码阅读：优秀开源项目
- 实战练习：复刻知名产品功能、造轮子

## 二、面试快速提升方案（4周计划）

### 2.1 Week 1：基础巩固

**每天 6 小时安排**：
- 3 小时：刷手写题（防抖、Promise 等）
- 1 小时：看 React 原理文章
- 2 小时：算法题（2 道）

### 2.2 Week 2：项目梳理

**任务清单**：
- 整理 2-3 个项目的 STAR 描述
- 提炼每个项目的 3 个技术亮点
- 准备项目中的难点和解决方案
- 继续每天 2 道算法题

### 2.3 Week 3：高频题突破

**任务清单**：
- 刷面试题库（牛客、掘金）
- 每天模拟 3 个面试题
- 整理个人答案模板
- 继续每天 2 道算法题

### 2.4 Week 4：实战演练

**任务清单**：
- 每天模拟面试 1 次
- 录音复盘
- 调整话术和节奏
- 复习薄弱环节

## 三、高频考点突破

### 3.1 JavaScript 核心（必考）

#### 闭包、作用域、this
- 手写 `call/apply/bind`
- 箭头函数 vs 普通函数
- 闭包应用场景（防抖、节流、私有变量）

#### 异步编程
- Promise 原理 + 手写 Promise
- async/await 原理
- Event Loop（宏任务/微任务）
- 并发控制（Promise.all/race/allSettled）

#### 原型链与继承
- 原型链图示
- 6 种继承方式
- `instanceof` 原理

#### 常见手写题（必练）

必须能默写的题目：
1. 防抖/节流
2. 深拷贝
3. 数组去重/扁平化
4. Promise.all/race
5. 函数柯里化
6. 发布订阅模式
7. LRU 缓存
8. 手写 bind

### 3.2 React 核心

#### 必答问题
- React 18 新特性（Concurrent、Suspense、Transition）
- Fiber 架构解决了什么问题
- Hooks 原理（为什么不能在条件语句中使用）
- diff 算法（key 的作用）
- 性能优化（useMemo/useCallback/React.memo）
- 状态管理方案选型

#### 实战题
- 手写简易版 useState/useEffect
- 实现虚拟列表
- 实现无限滚动
- 实现拖拽排序

### 3.3 Vue 核心

#### 关键点
- Vue 2 vs Vue 3 差异
- 响应式原理（defineProperty vs Proxy）
- Composition API 优势
- nextTick 原理

### 3.4 TypeScript

#### 高频问题
- 泛型应用
- 类型工具（Partial/Required/Pick/Omit）
- type vs interface
- 类型体操常见题

### 3.5 算法突击

#### 必刷题型（各 5-10 题）

**数组**：
- 双指针（三数之和、接雨水）
- 滑动窗口（最长无重复子串）
- 前缀和

**链表**：
- 反转链表
- 环形链表
- 合并链表

**树**：
- 二叉树遍历（前中后序）
- 二叉树深度/路径
- 二叉搜索树

**动态规划**（简单题）：
- 爬楼梯
- 最大子数组和
- 打家劫舍

**每日计划**：
- 每天 2-3 题
- 做完必须总结模板
- 重点是解题思路，不是死记硬背

## 四、项目经验包装（STAR 法则）

### 4.1 项目描述模板

#### Situation（背景）
- 什么业务场景
- 技术栈选型

#### Task（任务）
- 你的职责
- 要解决的核心问题

#### Action（行动）
- 技术方案设计
- 遇到的难点 + 如何解决
- 做了哪些优化

#### Result（结果）
- 性能提升 XX%
- 开发效率提升 XX%
- 具体数据指标

### 4.2 必须准备的亮点

**至少 3 个技术难点**：
- 性能优化（首屏加载从 5s 降到 2s）
- 复杂业务逻辑（多角色权限、复杂表单）
- 工程化实践（组件库、CI/CD、监控体系）

## 五、面试技巧

### 5.1 回答问题的结构

**技术问题模板**：
1. 简短定义（是什么）
2. 核心原理（为什么）
3. 应用场景（怎么用）
4. 注意事项（坑点）
5. 实际案例（项目经验）

#### 示例：闭包是什么？

❌ **差的回答**：
```
"闭包就是函数可以访问外部变量"
```

✅ **好的回答**：
```
"闭包是指函数能够访问其词法作用域外的变量。

原理上，JavaScript 引擎会保留外层函数的变量环境，
即使外层函数已经执行完毕。

常见应用场景有：
1. 防抖节流的计时器保存
2. 模块化的私有变量
3. React Hooks 的状态保存

需要注意闭包可能导致内存泄漏，要及时清理引用。

在我们项目中，我用闭包实现了一个请求缓存机制，
缓存已请求过的数据，减少了 50% 的重复请求。"
```

### 5.2 不会的问题怎么办

**策略**：
1. 承认不了解（诚实）
2. 说出相关知识（延伸）
3. 表达学习意愿（态度）

#### 示例

**面试官**："说 WebAssembly"

**回答**：
```
"WebAssembly 我了解不多，只知道它是一种可以在浏览器运行的
二进制格式，性能接近原生。

我之前关注过 Figma 用它来处理图形计算，性能提升很明显。
这块确实是我的薄弱点，如果入职后有相关场景，我会系统学习。

不过我对性能优化比较有经验，在项目中做过虚拟列表、
Web Worker 等优化，性能提升了 XX%。"
```

### 5.3 自我介绍模板（1分钟版）

```
"您好，我是XX，X年前端经验，主要技术栈是 React。

做过两个 Vue 项目，一个是物流结算系统，负责核心业务模块开发，
处理过复杂表单和多角色权限。

另一个是 uni-app 跨端项目，实现了 H5/小程序/APP 三端适配，
在性能优化方面做了虚拟滚动、条件编译等。

擅长性能优化和工程化，对 React 底层原理和前端架构比较感兴趣。
这次应聘贵司是因为XX（夸公司）。"
```

### 5.4 反问环节

#### 好问题
1. 团队技术栈和架构是怎样的？
2. 这个岗位主要负责什么业务方向？
3. 团队的 Code Review 和技术分享机制？
4. 有哪些技术挑战或者技术债需要解决？

#### 避免的问题
- 薪资福利（HR 面再问）
- 加班情况（显得消极）
- 太基础的问题（显得不专业）

## 七、核心建议

### 7.1 学习建议
1. **深度优先于广度**：先把 React 吃透，再扩展其他
2. **实践大于理论**：所有学习都要落地到项目
3. **持续输出**：博客/开源/分享，倒逼自己深入学习
4. **关注业务价值**：技术最终要服务业务
5. **保持好奇心**：关注新技术，但不盲目追新

### 7.2 面试建议
1. **80/20 法则**：80% 精力放在高频考点
2. **实战优先**：能手写 > 能说 > 知道
3. **项目为王**：所有技术点都往项目上靠
4. **控制时长**：每个问题 2-3 分钟，别滔不绝
5. **诚实态度**：不会就说不会，但要表现学习能力

### 7.3 心态建议
面试是个概率事件，被拒是正常的。多面几家找感觉，越面越好。
