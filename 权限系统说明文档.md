# 权限系统说明文档

## 一、权限系统概述

本项目采用基于菜单和按钮的权限控制系统，通过后端返回的权限数据动态生成路由和控制页面元素的显示。

**核心关键词**: `controlBtn` - 存储当前路由的权限信息，包括页面权限和按钮权限。

---

## 二、权限数据结构

### 2.1 权限数据来源

权限数据从后端接口获取，主要接口：

- **获取用户菜单**: `getMenu()` - 返回用户的菜单权限树
- **API 路径**: `src/api/public.js`

### 2.2 权限数据字段说明

后端返回的权限数据包含以下关键字段：

```javascript
{
  funcId: '',          // 功能ID
  funcName: '',        // 功能名称（中文）
  enfuncName: '',      // 功能英文名称（用于权限判断）
  actionName: '',      // 路由名称（如：userBase/workspace）
  url: '',             // 页面路径
  parentId: '',        // 父级ID
  hierarchy: '',       // 层级（1-顶级菜单，2-二级菜单，3-页面，4-按钮）
  funcType: '',        // 功能类型（'1'-页面，'2'-按钮）
  styleName: '',       // 按钮权限标识（如：btnAddCustomer）
  funcOption: '',      // 功能选项
  sortno: '',          // 排序号
  children: []         // 子权限
}
```

## 三、权限定义流程

### 3.1 登录后权限初始化

**流程位置**: `src/router/index.js` - `loadMenus()` 函数

```javascript
// 1. 获取用户信息
await store.dispatch('app/getUserInfo', {})

// 2. 获取用户菜单权限
const params = await store.dispatch('app/getUserMenu')
const { tree, datas } = params

// 3. 根据权限数据生成路由
testRouter(tree, datas, next, to)
```

### 3.2 权限数据处理

**处理位置**: `src/router/index.js` - `filterAsyncRouter()` 函数

**处理步骤**:

1. **过滤页面权限**: 遍历权限数据，筛选出有 `actionName` 和 `url` 的页面权限
2. **生成路由配置**: 为每个页面生成路由对象
3. **提取按钮权限**: 将按钮权限（`funcType === '2'`）添加到路由的 `meta` 中
4. **动态注册路由**: 使用 `router.addRoutes()` 动态添加路由

**核心代码逻辑**:

```javascript
// 遍历权限数据
Object.keys(routerHash).forEach(name => {
  const v = routerHash[name]
  const routerParams = {}
  
  // 设置路由基本信息
  routerParams.path = `${v.actionName.replace('userBase/', '')}`
  routerParams.name = v.actionName
  routerParams.component = loadView(component)
  
  // 设置路由 meta（包含权限信息）
  routerParams.meta = {
    title: v.funcName,
    funcId: v.funcId,
    parentId: v.parentId,
    hierarchy: v.hierarchy,
    enfuncName: v.enfuncName,
    funcOption: v.funcOption,
    funcType: v.funcType
  }
  
  // 提取按钮权限到 meta
  if (v.children && v.children.length > 0) {
    v.children.forEach(cv => {
      if (String(cv.funcType) === '2') {
        // 按钮权限标识作为 meta 的属性，值为 true
        routerParams.meta[cv.styleName] = true
      }
    })
  }
  
  params.children.push(routerParams)
})
```

### 3.3 权限存储

**存储位置**: Vuex Store - `src/vuex/modules/app.js`

**存储的权限数据**:

```javascript
state: {
  menu: null,              // 菜单树结构
  hash: null,              // 菜单哈希表（快速查找）
  oldMenu: null,           // 原始菜单数据
  controlBtn: {},          // 当前路由的权限信息（核心）
  organizationControl: {}  // 组织机构权限
}

mutations: {
  // 设置当前路由的权限信息
  SET_CONTROL_BTN(state, controlBtn) {
    state.controlBtn = controlBtn || {}
  }
}
```

### 3.4 路由切换时更新权限

**更新位置**: `src/router/index.js` - `router.afterEach()`

```javascript
router.afterEach((to) => {
  NProgress.done()
  
  // 将当前路由的 meta 信息存储到 controlBtn
  // meta 中包含了该页面的所有按钮权限
  store.commit('app/SET_CONTROL_BTN', to.meta)
})
```

## 四、权限使用方式

### 4.1 在组件中获取权限

**方式一：通过 Vuex 获取**

```javascript
import { mapState } from 'vuex'

export default {
  computed: {
    ...mapState('app', ['controlBtn'])
  }
}
```

**方式二：通过 $route.meta 获取**

```javascript
export default {
  computed: {
    hasAddPermission() {
      return this.$route.meta.btnAddCustomer
    }
  }
}
```

### 4.2 在模板中控制元素显示

**方式一：使用 v-if 判断**

```vue
<template>
  <!-- 通过 controlBtn 判断 -->
  <el-button v-if="controlBtn.btnAddCustomer" type="primary" @click="handleAdd">
    新增
  </el-button>
  
  <!-- 通过 $route.meta 判断 -->
  <el-button v-if="$route.meta.btnEditCustomer" type="primary" @click="handleEdit">
    编辑
  </el-button>
</template>

<script>
export default {
  computed: {
    controlBtn() {
      return this.$store.state.app.controlBtn
    }
  }
}
</script>
```

**方式二：使用 v-permission 指令**

项目提供了自定义指令 `v-permission` 用于权限控制。

**指令定义位置**: `src/directive/index.js`

```javascript
// 控制显隐 v-permission="'editbtn'" 或 v-permission="['editbtn','addbtn']"
Vue.directive('permission', {
  inserted(el, binding, vnode) {
    if (binding.value) {
      const toBeChecked = [].concat(binding.value)
      const meta = vnode?.context?.$router?.currentRoute?.meta || {}
      
      // 传入多个权限时，命中一个权限就显示
      const hasAuth = toBeChecked.some(item => !!meta[item])
      
      if (!hasAuth) {
        // 没有权限则移除元素
        el.parentNode && el.parentNode.removeChild(el)
      }
    }
  }
})
```

**使用示例**:

```vue
<template>
  <!-- 单个权限 -->
  <el-button v-permission="'btnAddCustomer'" type="primary">
    新增
  </el-button>
  
  <!-- 多个权限（有一个即可） -->
  <el-button v-permission="['btnEditCustomer', 'btnUpdateCustomer']" type="primary">
    编辑
  </el-button>
</template>
```

### 4.3 在 JavaScript 中判断权限

```javascript
export default {
  methods: {
    handleOperation() {
      // 方式一：通过 controlBtn
      if (this.controlBtn.btnDeleteCustomer) {
        this.deleteCustomer()
      }
      
      // 方式二：通过 $route.meta
      if (this.$route.meta.btnDeleteCustomer) {
        this.deleteCustomer()
      }
    }
  },
  
  computed: {
    controlBtn() {
      return this.$store.state.app.controlBtn
    }
  }
}
```

## 八、权限使用最佳实践

### 8.1 推荐做法

1. **优先使用 controlBtn**: 在组件中通过 `controlBtn` 判断权限，代码更清晰
   ```vue
   <el-button v-if="controlBtn.btnAdd" type="primary">新增</el-button>
   ```

2. **使用 v-permission 指令**: 对于简单的显隐控制，使用指令更简洁
   ```vue
   <el-button v-permission="'btnAdd'" type="primary">新增</el-button>
   ```

3. **多权限判断**: 当需要多个权限时，使用数组形式
   ```vue
   <el-button v-permission="['btnEdit', 'btnUpdate']">编辑</el-button>
   ```

### 8.2 注意事项

1. **权限标识命名**: 按钮权限标识必须以 `btn` 开头，使用驼峰命名
2. **权限粒度**: 按钮权限应该细化到具体操作，避免过于宽泛
3. **权限缓存**: `controlBtn` 会在路由切换时自动更新，无需手动刷新
4. **权限失效**: 如果权限数据变更，需要重新登录才能生效

### 8.3 常见问题

**Q1: 为什么按钮不显示？**

A: 检查以下几点：
- 后端是否返回了该按钮权限
- 权限标识是否正确（区分大小写）
- `controlBtn` 是否正确获取

**Q2: 如何调试权限？**

A: 在浏览器控制台执行：
```javascript
// 查看当前路由的权限
console.log(this.$route.meta)

// 查看 controlBtn
console.log(this.$store.state.app.controlBtn)

// 查看所有菜单权限
console.log(this.$store.state.app.menu)
```

**Q3: 权限数据何时更新？**

A: 权限数据在以下时机更新：
- 用户登录时
- 页面刷新时
- 路由切换时（更新 controlBtn）

## 九、权限扩展

### 9.1 添加新的按钮权限

1. **后端配置**: 在权限管理系统中添加新的按钮权限，设置 `funcType = '2'` 和 `styleName`
2. **前端使用**: 直接在组件中使用新的权限标识
   ```vue
   <el-button v-if="controlBtn.btnNewFeature">新功能</el-button>
   ```

### 9.2 自定义权限指令

如果需要更复杂的权限判断，可以扩展 `v-permission` 指令：

```javascript
// src/directive/index.js
Vue.directive('permission-and', {
  inserted(el, binding, vnode) {
    if (binding.value) {
      const toBeChecked = [].concat(binding.value)
      const meta = vnode?.context?.$router?.currentRoute?.meta || {}
      
      // 所有权限都必须满足
      const hasAuth = toBeChecked.every(item => !!meta[item])
      
      if (!hasAuth) {
        el.parentNode && el.parentNode.removeChild(el)
      }
    }
  }
})
```

使用示例：
```vue
<!-- 需要同时拥有两个权限 -->
<el-button v-permission-and="['btnEdit', 'btnAudit']">编辑并审核</el-button>
```

## 十、总结

本项目的权限系统通过以下方式实现：

1. **后端返回权限数据** → 包含菜单和按钮权限
2. **前端动态生成路由** → 根据权限数据生成可访问的路由
3. **提取按钮权限到 meta** → 将按钮权限存储在路由的 meta 中
4. **路由切换更新 controlBtn** → 每次路由切换时更新当前页面的权限信息
5. **组件中使用权限** → 通过 `controlBtn` 或 `v-permission` 指令控制元素显示

**核心关键词 `controlBtn`** 是权限系统的核心，它存储了当前路由的所有权限信息，包括页面权限和按钮权限。
